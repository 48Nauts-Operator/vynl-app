Vynl DB inspection

Container has no sqlite3 CLI; use node + better-sqlite3. Default DB paths above.

Find the actual DB path in the container

cosmos shell
docker exec vynl sh -c 'echo "VYNL_DB_DIR=${VYNL_DB_DIR:-(cwd)}"; ls -la "${VYNL_DB_DIR:-/app}/vynl.db" 2>/dev/null || find / -name "vynl.db" -not -path "/proc/*" 2>/dev/null'

List all app_settings (Spotify creds, AcoustID, etc.)

cosmos shell · production
docker exec vynl node -e "
const db = require('better-sqlite3')('/app/data/vynl.db',{readonly:true});
console.log(db.prepare('SELECT key, length(value) AS len FROM app_settings ORDER BY key').all());
"

Same, but local dev DB

Mac · local dev
node -e "
const db = require('better-sqlite3')('./vynl.db',{readonly:true});
console.log(db.prepare('SELECT key, length(value) AS len FROM app_settings ORDER BY key').all());
"

API keys via the /api/settings/keys endpoint

When the UI Save button silently drops the write (bug, task #81), set keys directly via curl. Don't paste secrets to a chat — paste them straight into the command on cosmos.

Read current state (masks secrets)

cosmos shell
curl -sS http://localhost:3101/api/settings/keys | python3 -m json.tool

Set a key (server reads DB first, env fallback)

cosmos shell
curl -sS -X PUT http://localhost:3101/api/settings/keys \
  -H "Content-Type: application/json" \
  -d '{"key":"spotifyClientId","value":"<real_id_32_chars>"}'

curl -sS -X PUT http://localhost:3101/api/settings/keys \
  -H "Content-Type: application/json" \
  -d '{"key":"spotifyClientSecret","value":"<real_secret_32_chars>"}'

curl -sS -X PUT http://localhost:3101/api/settings/keys \
  -H "Content-Type: application/json" \
  -d '{"key":"spotifyRedirectUri","value":"https://cosmos.tail138398.ts.net/api/spotify/callback"}'

Clear a key (revert to env fallback)

cosmos shell
curl -sS -X DELETE "http://localhost:3101/api/settings/keys?key=spotifyClientId"

Registered keys: anthropic, replicate, acoustid, spotifyClientId, spotifyClientSecret, spotifyRedirectUri, youtube.

Wishlist cleanup

The 7-phase extract used to dump every unmatched Spotify track into wish_list with type='spotify_missing'. Clean up leftovers.

Inspect first

cosmos / mac
node -e "
const db = require('better-sqlite3')('./vynl.db',{readonly:true});
const counts = db.prepare(\"SELECT type, COUNT(*) AS n FROM wish_list GROUP BY type\").all();
console.log('by type:', counts);
const recent = db.prepare(\"SELECT MIN(created_at) AS oldest, MAX(created_at) AS newest FROM wish_list\").get();
console.log('range:', recent);
"

Nuke leftover spotify_missing only (keep today's curated rows)

cosmos / mac
node -e "
const db = require('better-sqlite3')('./vynl.db');
const r = db.prepare(\"DELETE FROM wish_list WHERE type='spotify_missing' AND created_at < date('now')\").run();
console.log('deleted:', r.changes);
"

Nuke ALL wishlist rows

cosmos / mac · destructive
node -e "
const db = require('better-sqlite3')('./vynl.db');
const before = db.prepare('SELECT COUNT(*) AS n FROM wish_list').get();
const r = db.prepare('DELETE FROM wish_list').run();
console.log('before:', before.n, '· deleted:', r.changes);
"
In container: swap ./vynl.db for /app/data/vynl.db and prefix with docker exec vynl.

Lidarr cleanup

When the Lidarr push goes wide (the 122 GB / 73 album disaster path), use these to unwind. Need your Lidarr URL + API key (Lidarr → Settings → General → Security).

Variables (set once per shell)

cosmos shell
LIDARR=http://192.168.74.79:8686
KEY=<your_api_key>

Inspect: how many artists / missing albums?

cosmos shell
echo "ARTISTS:"
curl -sS -H "X-Api-Key: $KEY" "$LIDARR/api/v1/artist" \
  | python3 -c "import json,sys; a=json.load(sys.stdin); print(len(a),'artists'); [print(f' {x[\"id\"]:>5}  {x[\"artistName\"]}') for x in a[:20]]"

echo
echo "MISSING ALBUMS (Wanted list):"
curl -sS -H "X-Api-Key: $KEY" "$LIDARR/api/v1/wanted/missing?pageSize=5&monitored=true" \
  | python3 -c "import json,sys; r=json.load(sys.stdin); print(r['totalRecords'],'missing total')"

Empty the Wanted/Missing list (unmonitor, keep artists + files)

cosmos shell
ids=$(curl -sS -H "X-Api-Key: $KEY" "$LIDARR/api/v1/wanted/missing?pageSize=5000&monitored=true" \
  | python3 -c "import json,sys; r=json.load(sys.stdin)['records']; print(','.join(str(a['id']) for a in r))")

# bulk unmonitor (single PUT)
curl -sS -X PUT -H "X-Api-Key: $KEY" -H "Content-Type: application/json" \
  "$LIDARR/api/v1/album/monitor" \
  -d "{\"albumIds\":[$ids],\"monitored\":false}" | head -c 200

# confirm
curl -sS -H "X-Api-Key: $KEY" "$LIDARR/api/v1/wanted/missing?monitored=true" \
  | python3 -c "import json,sys; print(json.load(sys.stdin)['totalRecords'],'still missing')"
Lidarr's SQLite can lock up when its background "Search for Missing" task is running. First request can take 30–90 s. If it stays blinking past ~2 min, Ctrl+C and check Lidarr UI → System → Tasks.

Delete all artists (files stay on disk)

cosmos shell · destructive
curl -sS -H "X-Api-Key: $KEY" "$LIDARR/api/v1/artist" \
  | python3 -c "import json,sys; print('\n'.join(str(x['id']) for x in json.load(sys.stdin)))" \
  | xargs -I{} curl -sS -X DELETE -H "X-Api-Key: $KEY" "$LIDARR/api/v1/artist/{}?deleteFiles=false&addImportListExclusion=true"

curl -sS -H "X-Api-Key: $KEY" "$LIDARR/api/v1/artist" | python3 -c "import json,sys; print(len(json.load(sys.stdin)),'remaining')"

Lidarr UI cheat — same things, slower but clearer

  • Wanted → Missing: bump page size → select all → Unmonitor Selected.
  • Library → Artists: Options → Mass Editor → select all → Delete (uncheck "Delete artist files", check "Add list exclusion").
  • Activity → Queue: remove anything mid-grab; "Remove from download client" if you don't want it.

Sonos diagnostics

v0.6.22+ logs every Sonos request as [sonos <reqId>] .... Watch in Dozzle or via docker logs.

Tail Sonos lines as you Play

cosmos shell
docker logs -f vynl 2>&1 | grep -E '\[sonos|UPnP'

Verify env IP matches cosmos's actual IP

cosmos shell
docker exec vynl env | grep VYNL_HOST    # what Vynl sees
hostname -I                              # what cosmos actually is
Failure mode lookup:
  • Sonos app "Unable to connect to <IP>" → URL host unreachable. Check env vs actual IP.
  • UPnPError 701 (Transition not available) → same root cause; speaker rejected Play.

Tailscale HTTPS (for Spotify OAuth)

Spotify dropped http:// redirect URIs (except loopback) in Feb 2026. Tailscale Serve gives a public-Let's-Encrypt-signed cert for the tailnet hostname, no public exposure.

Start (persistent, survives container restarts)

cosmos shell
docker exec tailscale tailscale serve --bg --https=443 http://localhost:3101
docker exec tailscale tailscale serve status

If "foreground listener already exists for port 443"

cosmos shell
docker exec tailscale tailscale serve reset
docker exec tailscale tailscale serve --bg --https=443 http://localhost:3101
# or restart the container if reset doesn't clear:
docker restart tailscale && docker exec tailscale tailscale serve --bg --https=443 http://localhost:3101

Stop

cosmos shell
docker exec tailscale tailscale serve --https=443 off
docker exec tailscale tailscale serve reset    # nukes all serve config

Result URL: https://cosmos.tail138398.ts.net/ — only reachable from the tailnet.

Dozzle (browser log viewer)

Lightweight web UI for any container's logs on the host. Used to share live logs across debugging.

Portainer stack YAML

portainer · new stack
version: "3.8"
services:
  dozzle:
    image: amir20/dozzle:latest
    container_name: dozzle
    restart: unless-stopped
    environment:
      - DOZZLE_ADDR=:8181        # change internal port (default 8080 collided)
      - DOZZLE_LEVEL=info
      - DOZZLE_NO_ANALYTICS=true
    volumes:
      - /var/run/docker.sock:/var/run/docker.sock:ro
    ports:
      - "8181:8181"

Find a free port on cosmos

cosmos shell
for p in 8181 8800 8444 7777 9090; do
  ss -ltn "( sport = :$p )" | tail -n +2 | grep -q . || { echo "FREE: $p"; break; }
done

URLs once running: http://192.168.74.79:8181/ or http://100.119.120.45:8181/.

Release flow

Forgejo development → merge into UAT → Forgejo Action mirrors to GitHub main → docker-publish.yml builds cand0rian/vynl-app:vX.Y.Z + :latest on Docker Hub + GHCR.

Bump + push

Mac · repo root
# 1. Edit package.json version manually, then:
git add package.json && git commit -m "chore(release): vX.Y.Z — summary"
git push forgejo development

# 2. Merge into UAT (triggers release workflow)
git checkout UAT
git merge --no-ff development -m "chore(uat): release vX.Y.Z — summary"
git push forgejo UAT
git checkout development

Watch for build completion

Mac
# Replace X.Y.Z with the version you just released
until tag=$(curl -sS "https://hub.docker.com/v2/repositories/cand0rian/vynl-app/tags/vX.Y.Z/" 2>/dev/null \
  | python3 -c "import json,sys; print(json.load(sys.stdin).get('name','none'))" 2>/dev/null); [ "$tag" = "vX.Y.Z" ]; do sleep 60; done
echo "Docker Hub :vX.Y.Z LIVE"
Cosmos pulls cand0rian/vynl-app:latest by default — Portainer stack "Update" with "Re-pull image" fetches the new build. Pin to :vX.Y.Z in the stack if you want a specific version.
